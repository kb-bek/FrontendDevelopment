Негизги бет IP протоколу Жабдуулар OSI модели

2-вариант

Суроо: OSI моделинин үчүнчү катмарын түшүндүрүп бериңиз. Бул катмарда кайсы шаймандар иштейт жана кайсы протокол колдонулат?

Жооп:

OSI моделинин үчүнчү катмары, ошондой эле Network Layer деп аталат, ар кандай тармактардын ортосунда маалыматтарды өткөрүүдө негизги ролду ойнойт. Бул деңгээлде маалыматтардын маршрутизациясы жана тармак ресурстарын башкаруу жүргүзүлөт. Бул жерде OSI Layer 3 жана ага байланыштуу инструменттер/протоколдордун кээ бир маанилүү аспектилери бар:

  1. Маршрутизация: 3-кабаттын негизги функцияларынын бири ар кандай тармактардын ортосунда маалыматтарды багыттоо болуп саналат. Маршрутизаторлор (роутерлор) бул катмарда иштешет жана маалыматтарды жөнөтүүчүдөн кабыл алуучуга ар кандай тармактар ​​аркылуу кантип өткөрүү жөнүндө чечим кабыл алышат.

  2. IP протоколдору: IP (Internet Protocol) үчүнчү катмардагы эң кеңири таралган протокол. IP даректер тармактагы түзмөктөрдү аныктоо үчүн колдонулат, ал эми IP протоколу бул түзмөктөр кантип байланышып, маалыматтардын багытын аныктайт.

  3. ICMP (Internet Control Message Protocol): ICMP ката билдирүүлөрүн жөнөтүү жана тармак абалын көзөмөлдөө үчүн колдонулат. Мисалы, ал "ping" жана "traceroute" билдирүүлөрүн иштетүү үчүн колдонулат.

  4. OSPF (Open Shortest Path First): Бул роутерлердин ортосундагы эң жакшы маршруттарды аныктоо үчүн автономдуу системаларда (AS) колдонулган маршрутташтыруу протоколу. OSPF ички маршруттук протоколдордун бири болуп саналат.

  5. BGP (Border Gateway Protocol): BGP Интернетте ар кандай автономдуу системалар (AS) ортосунда колдонулган маршруттук протоколу болуп саналат. Ал ар кандай АСлардын ортосундагы каттамдарды аныктайт жана эл аралык протокол болуп саналат.

  6. CIDR (Classless Inter-domain Routing): CIDR IP даректерин бириктирүү жана дарек мейкиндигин оптималдаштыруу үчүн колдонулат.

  7. VPN (Virtual Private Network): IPsec жана GRE сыяктуу VPN технологиялары 3-кабатта иштейт жана алыскы тармактардын ортосунда коопсуз жана купуя байланышты камсыз кылат.
